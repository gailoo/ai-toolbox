## 为什么不能只看模型榜单

模型能力很重要，但实际选型还要考虑产品体验、工作流集成、权限安全、团队协作、成本和可持续维护。

## 五个选型问题

1. **任务是什么？** 写作、代码、搜索、设计、数据分析还是自动化？
2. **输入是什么？** 文本、图片、代码库、表格、数据库还是网页？
3. **输出给谁用？** 个人、团队、客户、管理层还是系统 API？
4. **风险有多高？** 是否涉及隐私、资金、生产环境、客户承诺？
5. **是否需要持续更新？** 一次性工具和长期系统的选型完全不同。

## 推荐方法

用“三层评估”：

- **模型层**：准确性、推理、多模态、速度。
- **产品层**：交互、集成、协作、稳定性。
- **业务层**：成本、权限、安全、ROI、可维护性。

## 常见结论

- 日常对话：ChatGPT / Claude / Gemini。
- 代码开发：Cursor / CodeBuddy / Copilot。
- 快速 UI：v0 / Bolt.new / Lovable。
- 企业工作流：WorkBuddy / Dify / Coze / n8n。
- 知识问答：RAG 平台 + 可引用来源的搜索工具。
