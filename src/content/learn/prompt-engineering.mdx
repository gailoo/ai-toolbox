## 一个好 Prompt 的基本结构

推荐使用这个结构：

1. **角色**：你希望 AI 以什么身份工作。
2. **目标**：你要完成什么任务。
3. **上下文**：背景、资料、限制条件。
4. **输出格式**：表格、清单、报告、JSON、代码等。
5. **评价标准**：什么算好，什么不接受。

## 示例

```text
你是一个面向B端增长产品经理的AI产品研究助手。
请基于以下材料，整理竞品AI工具的分类、核心能力、适用场景和局限性。
输出为Markdown表格，避免泛泛而谈，每个结论都要对应证据。
```

## 常见错误

- 只说“帮我分析一下”，没有目标和格式。
- 让模型凭空生成数据。
- 不说明受众、场景和判断标准。
- 一次塞太多目标，导致输出散乱。

## 更好的做法

把复杂任务拆成多轮：

1. 先让模型定义问题。
2. 再列出分析框架。
3. 再填充信息。
4. 最后压缩成可交付版本。
